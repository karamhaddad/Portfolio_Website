Prefetcher Simulations for Row Hammer Attacks

Prefetching

Prefetching is a technique that loads data into the cache before it’s needed. By predicting future data requirements, prefetching improves program execution speed.
Figure 1: Prefetchers explained [1]

Row Hammer

Row Hammer is a DRAM vulnerability where frequent access to a memory row causes bit flips in adjacent rows. This issue is worsened by prefetchers, as repeated access to specific rows increases the risk of data corruption.
Figure 2: Cache layout [2]

Traces

A trace is a collection of data access patterns. Different prefetchers exploit different patterns, which may range from completely random to highly predictable.
Figure 3: Texas A&M Trace Speedup Survey [3]

T-SKID: Timing-SKID Prefetcher

The T-SKID prefetcher, which I implemented, is designed to delay prefetching until the optimal moment. This strategic delay prevents unnecessary cache misses and improves cache efficiency.

Design

  • Implemented core data structures:
    • Target Table (TT)
    • Address Prediction Table (APT)
    • Inflight Prefetch Table (IPT)
    • Recent Request Program Counter Queue (RRPCQ)
  • Followed the research paper’s methodology for learning addresses, timing, and issuing prefetches.


Figure 4: T-SKID Issuing a prefetch (a), and learning timing (b) [6]

Results

  • IPC Speedup: Significant improvement in patterned traces.
  • Randomized traces like MCF showed limited gains, as expected.
  • Accuracy: High accuracy in workloads with discernible patterns.
  • Hammer Rate: Managed row activations effectively, aligning with expected behavior from the research.

Team Contributions

Beyond T-SKID, the team implemented and validated additional prefetchers:

  • LSTM-based Prefetcher (Nathaniel Bush): A neural network to predict memory access offsets.
  • Tag Correlating Prefetcher (John Iler): Utilized tag sequences to predict cache accesses.
  • Temporal Prefetcher (Garvit Dhingra): A structure-based temporal approach using custom caches.

Integrated Results

The team combined results to analyze:

  • Speedup: Achieved measurable IPC improvements over benchmarks.
  • Accuracy: Evaluated useful vs. useless prefetches.
  • Hammer Rate: Compared row activations to a baseline (no prefetching).
